משימה 2 – אילו ציונים היו מעל הממוצע?

עכשיו רוצים להוסיף: הדפסת כל הציונים שהיו מעל הממוצע.

🤔 האם אפשר לבדוק כל ציון מול הממוצע כבר בלולאה הראשונה – זו שקולטת את הציונים ומחשבת את הסכום?
לא! כדי לחשב את הממוצע צריך לסיים לקלוט את כל הציונים קודם. כל עוד לא סיימנו לקלוט – אין לנו ממוצע להשוות אליו.
אז מה עושים?
🤔 אם אי אפשר לבדוק בלולאה הראשונה – אז מה הפיתרון?
צריך לעבור שוב על הציונים! אחרי שחישבנו את הממוצע – נבקש מהמשתמש שוב להזין את הציונים, ונשווה כל אחד לממוצע.
משימה 3 – מימוש

הוסיפו ללולאה שנייה הקולטת שוב את הציונים ומדפיסה רק את אלה שמעל הממוצע.

// לולאה ראשונה - קליטה וחישוב ממוצע int sum = 0; for (int i = 0; i < 5; i++) { Console.Write("Enter grade " + (i + 1) + ": "); int grade = int.Parse(Console.ReadLine()); sum = sum + grade; } double average = (double)sum / 5; Console.WriteLine("Average: " + average); // לולאה שנייה - קליטה שנייה והשוואה לממוצע Console.WriteLine("Grades above average:"); for (int i = 0; i < 5; i++) { Console.Write("Enter grade " + (i + 1) + " again: "); int grade = int.Parse(Console.ReadLine()); if (grade > average) { Console.WriteLine(grade); } }
רגע, עצרו
🤔 אתם המשתמש. הרגע הזנתם 5 ציונים. ועכשיו התוכנית מבקשת מכם להזין אותם שוב... איך זה מרגיש?
זו חוויית משתמש גרועה. המשתמש כבר הזין את הנתונים – לבקש שוב זה מעצבן ולא מקצועי. ובעצם – הנתונים כבר היו לנו. פשוט לא שמרנו אותם.

ועכשיו שאלה: מה עושים כאשר הקלט לא מגיע מהשתמש?
🤔 דמיינו שהנתונים לא מגיעים ממשתמש אלא נוצרים אוטומטית – כמו מדידה שמתבצעת בזמן אמת. האם גם אז אפשר לבקש "שוב"?
💭 מה אם במקום ציונים, היינו מודדים משהו שמגיע בזמן אמת – למשל, קצב הלב שלכם במהלך אימון כושר? חיישן מודד כל דקה ושולח את התוצאה – ואי אפשר לחזור ולמדוד שוב.

משימה 3 – דקות מעל הממוצע

עכשיו רוצים לדעת: באילו דקות הדופק היה מעל הממוצע? אלו היו הדקות שבהן עבדתם הכי קשה.

🤔 האם אפשר לבדוק כל מדידה מול הממוצע בלולאה שכבר כתבתם? למה כן או למה לא?
לא – אותה בעיה כמו בחלק א׳. כדי לבדוק מול הממוצע, צריך קודם לסיים את כל המדידות ולחשב את הממוצע. אי אפשר לדעת את הממוצע באמצע.
אז מה עושים?
🤔 בחלק א׳ פתרנו את זה על ידי בקשה שוב מהמשתמש. האם אפשר לעשות את אותו דבר כאן?
לא! המדידות נוצרות אקראית – כל פעם שנריץ שוב את הלולאה נקבל מספרים שונים לגמרי. אין "משתמש", והחיישן לא חוזר לאותה הדקה. הנתונים פשוט... נעלמים
פיתרון – נשמור כל מדידה במשתנה

רעיון: מה אם נשמור כל מדידה במשתנה? bpm1, bpm2... עד bpm10. אז הנתונים לא יעלמו!

כתבו מחדש את התוכנית – הפעם ללא לולאה לקליטה. הגדירו 10 משתנים נפרדים, שמרו בכל אחד מדידה אקראית, חשבו ממוצע, והדפיסו את המדידות שמעל הממוצע.

Random rnd = new Random(); int bpm1 = rnd.Next(60, 181); int bpm2 = rnd.Next(60, 181); int bpm3 = rnd.Next(60, 181); int bpm4 = rnd.Next(60, 181); int bpm5 = rnd.Next(60, 181); int bpm6 = rnd.Next(60, 181); int bpm7 = rnd.Next(60, 181); int bpm8 = rnd.Next(60, 181); int bpm9 = rnd.Next(60, 181); int bpm10 = rnd.Next(60, 181); double average = (double)(bpm1 + bpm2 + bpm3 + bpm4 + bpm5 + bpm6 + bpm7 + bpm8 + bpm9 + bpm10) / 10; Console.WriteLine("Average BPM: " + average); Console.WriteLine("Minutes above average:"); if (bpm1 > average) Console.WriteLine(bpm1); if (bpm2 > average) Console.WriteLine(bpm2); // ... וכן הלאה עד bpm10
אתגר – שנו את התוכנית לאימון של שעה

האימון עכשיו הוא שעה שלמה – 60 דקות. שנו את התוכנית כך שתעבוד עבור 60 מדידות.

כך ייראה הקוד עם 60 משתנים:
int bpm1 = rnd.Next(60, 181); int bpm2 = rnd.Next(60, 181); int bpm3 = rnd.Next(60, 181); // ... עוד 54 שורות כאלה... int bpm58 = rnd.Next(60, 181); int bpm59 = rnd.Next(60, 181); int bpm60 = rnd.Next(60, 181); double average = (double)(bpm1 + bpm2 + bpm3 + ... + bpm60) / 60; // ועוד 60 בדיקות if...

חלק ג׳ – מערך
📁 צרו פרויקט חדש ב-Visual Studio. שם הקובץ: HeartRateArray.cs
הכירו את המערך

הנה קוד שיעזור לנו בהמשך לפתור את הבעיה שלנו – הוא מודד דופק אחת לדקה במשך 10 דקות, ומדפיס כל מדידה:

Random rnd = new Random(); int[] heartRate = new int[10]; for (int i = 0; i < heartRate.Length; i++) { heartRate[i] = rnd.Next(60, 181); Console.WriteLine("Minute " + (i + 1) + ": " + heartRate[i] + " BPM"); }

העתיקו את הקוד והריצו אותו.

🤔 מה עשתה התוכנית? מה הדפיסה? הריצו אותה כמה פעמים – מה שמתם לב?
שורה ראשונה – הגדרת המערך

הסתכלו על השורה:

int[] heartRate = new int[10];

זו הגדרה של מערך – כלי שמחזיק מספר ערכים מאותו טיפוס תחת שם אחד.

  • int[] – מערך של מספרים שלמים
  • heartRate – שם המערך
  • new int[10] – יוצרים מערך עם 10 תאים
🤔 מה לדעתכם יש בתאים מיד אחרי ההגדרה, לפני שמילאנו אותם?
כל התאים מכילים 0 – זו ברירת המחדל של int ב-C#.
גישה לתא לפי אינדקס

כל תא במערך מזוהה לפי אינדקס – מספר סידורי שמתחיל ב- 0.

הוסיפו את השורות הבאות אחרי הלולאה וראו מה קורה:

Console.WriteLine(heartRate[0]); // תא ראשון Console.WriteLine(heartRate[9]); // תא אחרון
🤔 מה האינדקס של התא האחרון במערך של 10 תאים? למה לא 10?
האינדקס האחרון הוא 9 – כי מתחילים מ-0. מערך של 10 תאים: אינדקסים 0 עד 9.
מה קורה אם יוצאים מהגבולות?

עכשיו נסו להוסיף שורה זו ולהריץ:

Console.WriteLine(heartRate[10]);
🤔 מה קרה? למה?
קיבלתם שגיאת ריצה: IndexOutOfRangeException – יציאה מגבולות המערך.
מערך של 10 תאים מאפשר גישה רק לאינדקסים 0 עד 9. אינדקס 10 לא קיים.
זו שגיאה נפוצה מאוד – חשוב לזכור: האינדקס האחרון הוא תמיד Length - 1.

כדי להמשיך – הוסיפו // לפני השורה הזו כדי להשבית אותה:
// Console.WriteLine(heartRate[10]);
התכונה Length וגודל קבוע

הוסיפו שורה זו לקוד וראו מה מדפיסה:

Console.WriteLine(heartRate.Length);
🤔 מה הדפיסה? למה שימושי להשתמש ב- heartRate.Length בלולאה במקום המספר 10?
אם נשנה את גודל המערך בעתיד – נשנה רק בהגדרה, והלולאה תתאים את עצמה אוטומטית.
חשוב לדעת: גודל המערך קבוע – אי אפשר להוסיף תאים אחרי שהמערך נוצר.
תחביר חלופי לאתחול

יש עוד דרך להגדיר מערך – כשהערכים ידועים מראש:

int[] heartRate = {72, 85, 91, 88, 76, 95, 102, 98, 87, 79};

זה יוצר מערך של 10 תאים עם הערכים שציינו – בשורה אחת.

🤔 באיזה מקרה תעדיפו את הדרך עם new int[10] ובאיזה מקרה את הדרך עם הסוגריים המסולסלים?
כשהערכים לא ידועים מראש (למשל – מגיעים מחיישן או מהמשתמש) נשתמש ב- new int[10] ונמלא בלולאה.
כשהערכים ידועים מראש – הסוגריים המסולסלים נוחים יותר.
חזרה לבעיה המקורית – ממוצע ומדידות מעל הממוצע

עכשיו יש לנו את כל הכלים. בואו נפתור את הבעיה שהתחלנו איתה – חישוב ממוצע דופק, ואז הדפסת המדידות שהם מעל הממוצע.

הוסיפו ללולאה הקיימת חישוב סכום, ואחריה – חשבו ממוצע והדפיסו אותו.

int sum = 0; for (int i = 0; i < heartRate.Length; i++) { sum = sum + heartRate[i]; } double average = (double)sum / heartRate.Length; Console.WriteLine("Average BPM: " + average);
המעבר השני – הפעם זה עובד!

עכשיו הוסיפו לולאה נוספת שעוברת על המערך ומדפיסה רק את המדידות שמעל הממוצע.

🤔 האם הפעם אפשר לעבור שוב על הנתונים? למה?
כן! כל המדידות שמורות במערך – אפשר לעבור עליהן כמה פעמים שרוצים. זו בדיוק הנקודה!
Console.WriteLine("Minutes above average BPM:"); for (int i = 0; i < heartRate.Length; i++) { if (heartRate[i] > average) { Console.WriteLine("Minute " + (i + 1) + ": " + heartRate[i] + " BPM"); } }
עוד פעולות שימושיות על מערכים

כעת כשיש לנו את המערך עם כל המדידות, אפשר לעשות עוד דברים מעניינים. הוסיפו לתוכנית:

א׳ – מצאו את מדידת הדופק הגבוהה ביותר:

🤔 איך מוצאים את הערך הגבוה ביותר במערך? מה צריך לאתחל לפני הלולאה?
int maxBpm = heartRate[0]; // מתחילים מהתא הראשון for (int i = 1; i < heartRate.Length; i++) { if (heartRate[i] > maxBpm) { maxBpm = heartRate[i]; } } Console.WriteLine("Max BPM: " + maxBpm);

ב׳ – ספרו כמה דקות הדופק היה מעל 150:

🤔 כתבו לולאה שסופרת כמה תאים במערך גדולים מ-150.
int count = 0; for (int i = 0; i < heartRate.Length; i++) { if (heartRate[i] > 150) { count++; } } Console.WriteLine("Minutes above 150 BPM: " + count);
מערכים מטיפוסים שונים

כל תאי המערך חייבים להיות מאותו טיפוס – לא ניתן לערבב טיפוסים שונים במערך אחד.

מערך יכול להיות מכל טיפוס נתונים. הנה לדוגמה מערך מטיפוס double שמכיל ממוצעי ציונים של תלמידים בכיתה. הקוד שלפנינו משנה את ערכם של תאי המערך ע"י הוספת 10 נקודות בונוס לממוצע של כל תלמיד.ה:

double[] averages = {78.5, 92.0, 65.3, 88.7, 71.4}; // add 10 bonus points to each student for (int i = 0; i < averages.Length; i++) { averages[i] = averages[i] + 10; } // print updated averages for (int i = 0; i < averages.Length; i++) { Console.WriteLine("Student " + (i + 1) + ": " + averages[i]); }

העתיקו והריצו. שימו לב: אנחנו יכולים לשנות את הערך בתא – לא רק לקרוא ממנו.

מערך של תווים

הנה דוגמה נוספת – מערך של תווים. העתיקו והריצו:

char[] myChars = {'h', 'e', 'l', 'l', 'o'}; for (int i = 0; i < myChars.Length; i++) { Console.Write(myChars[i]); } Console.WriteLine();

מה מדפיסה התוכנית?

רגע – את זה כבר ראיתם!

הסתכלו על שתי הדוגמאות הבאות זו לצד זו:

// מערך של תווים: char[] myChars = {'h', 'e', 'l', 'l', 'o'}; Console.WriteLine(myChars[0]); // h Console.WriteLine(myChars.Length); // 5 // מחרוזת - בדיוק אותו תחביר! string myString = "hello"; Console.WriteLine(myString[0]); // h Console.WriteLine(myString.Length); // 5

אותו אינדקס, אותו Length, אותה גישה לפי מיקום – בדיוק אותו תחביר.

מחרוזת היא בעצם מערך של תווים. כשלמדתם לעבור על מחרוזת עם לולאה, השתמשתם בדיוק באותם העקרונות – מבלי לדעת שאתם כבר עובדים עם מערך. 😊
